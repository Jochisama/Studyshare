TP 01

TFD & Filtrage Numérique de Base TP1

1. Lecture et écoute d'un signal audio

import numpy as np
import scipy.signal as sp
import scipy.io.wavfile as wav
import sounddevice as snd

fe, x = wav.read('signal.wav')   # fe = fréquence éch., x = échantillons
snd.play(x, fe); snd.wait()
Paramètres clés
  • fe = fréquence d'échantillonnage (Hz)
  • Te = 1/fe = période d'échantillonnage
  • N = len(x) = nombre d'échantillons
  • t = np.arange(0,N)*Te = vecteur temps
Types de données audio
  • int16 → entre −32768 et +32767
  • uint8 → entre 0 et 255
  • Toujours convertir en float pour les calculs
  • Reconvertir avant lecture/écriture

2. TFD (Transformée de Fourier Discrète)

Formule fondamentale : X[k] = Σ x[n]·e^(-j2πkn/N) pour k = 0, 1, ..., N-1

# TFD centrée (spectre bilatéral de -fe/2 à +fe/2)
NF = N                                      # nb points fréquentiels
TFx = np.fft.fft(x, NF) / N               # FFT normalisée
TFx = np.fft.fftshift(TFx)                # centrage autour de 0 Hz
TFxm = np.abs(TFx)                         # module (spectre amplitude)
TFxdB = 20 * np.log10(TFxm + 1e-12)       # en dB (+ epsilon anti-log(0))
f = np.arange(-NF/2, NF/2) * fe / NF     # vecteur fréquence centré
⚠

Pourquoi diviser par N ? Pour normaliser l'amplitude — sans ça, l'amplitude du spectre dépend du nombre de points. Avec /N, un signal sinusoïdal d'amplitude A donne un pic spectral d'amplitude A/2.

⚠

Pourquoi +1e-12 dans log10 ? Pour éviter log10(0) = -∞ qui fait planter le programme. C'est un epsilon mathématique négligeable.

3. Filtre numérique — Pôles, Zéros, Réponses

Passe-bas du 2ème ordre

H(z) = (z²+2z+1) / (z²−(√2/2)z+1/2)
b = [1, 2, 1]
a = [1, -0.5*np.sqrt(2), 0.5]

Zéros en z = −1 (double) → atténuation à f = fe/2

Passe-haut du 2ème ordre

H(z) = (z²−2z+1) / (z²−(√2/2)z+1/2)
b = [1, -2, 1]
a = [1, -0.5*np.sqrt(2), 0.5]

Zéros en z = +1 → atténuation à f = 0 Hz

# Réponse impulsionnelle (filtrer une impulsion de Dirac)
Lh = 50
Dirac = np.zeros(Lh); Dirac[0] = 1
h = sp.lfilter(b, a, Dirac)

# Réponse fréquentielle
freq, Hf = sp.freqz(b, a, 256, fs=fe)   # 256 = résolution

# Pôles et zéros
z, p = zplane(b, a)                       # de plot_zplane.py

4. Filtrage et formatage

# Filtrage
y = sp.lfilter(b, a, x)

# Formatage int16
n16 = 2**16
yf = np.int16((y - min(y)) * (n16-1) / (max(y) - min(y)) - n16//2)

# Formatage uint8
n8 = 2**8
yf = np.uint8((y - min(y)) * (n8-1) / (max(y) - min(y)))

Observation TP1 : Le filtre passe-bas donne un son "sourd/grave", le passe-haut donne un son "aigu/métallique". On entend clairement la différence car les bandes fréquentielles éliminées correspondent à des composantes perceptibles.

TP 02

Synthèse de Filtres Numériques RIF TP2

1. Théorie — Filtre RIF

Un filtre RIF (Réponse Impulsionnelle Finie) a A(z) = 1 → pas de pôles (sauf à z=0) → toujours stable. Sa réponse impulsionnelle est exactement les coefficients b[n].

H(z) = b₀ + b₁z⁻¹ + ... + bₙz⁻ᴺ

2. Gabarit et spécifications

Paramètre Symbole Définition
Fréquence passante fp Limite de la bande passante
Fréquence atténuée fa Début de la bande atténuée
Bande de transition Δf = fa − fp Zone entre passant et atténué
Fréquence de coupure fc = (fa + fp) / 2 Milieu de la bande de transition
Atténuation passante Ap = −20·log(1−δp) Ondulation en bande passante
Atténuation minimale Aa = −20·log(δa) Att. en bande atténuée

3. Table des fenêtres — À mémoriser !

Fenêtre Δfw/Fe Aa (dB) Formule N (approx.)
Rectangulaire 1.8/N 21 dB N ≈ 1.8·Fe/Δf
Bartlett 2/N 25 dB N ≈ 2·Fe/Δf
Hanning 6.2/N 44 dB N ≈ 6.2·Fe/Δf
Hamming 6.6/N 53 dB N ≈ 6.6·Fe/Δf
Blackman 11/N 74 dB N ≈ 11·Fe/Δf
💡

Règle d'or : Si Aa > 50 dB → fenêtre de Hamming. Si Aa > 74 dB → Blackman. L'examen fixe souvent Aa > 50 dB donc Hamming est la réponse.

4. Méthode des fenêtres — Les 4 étapes

ÉTAPE 1

Spécifications : Fixer fp, fa, Aa, Ap, Fe. Calculer fc = (fp+fa)/2, Δf = fa−fp

ÉTAPE 2

Choix fenêtre : Selon Aa requis, consulter la table. Aa > 50 → Hamming

ÉTAPE 3

Ordre du filtre : N ≈ K·Fe/Δf (K depuis table), choisir N impair → Ncoef = N

ÉTAPE 4

Calcul h(n) : ĥ(n) = h(n − (N−1)/2) · w(n) où h(n) = 2fc·sinc(2fc·n)

5. Code complet TP2

# ==== BRUIT : 3 sinusoïdes ====
A = 50
f1, f2, f3 = 1500, 2000, 3000
bruit = A * (np.sin(2*np.pi*f1*t) + np.sin(2*np.pi*f2*t) + np.sin(2*np.pi*f3*t))
x_bruite = x.astype(np.float64) + bruit

# ==== CONCEPTION FILTRE RIF ====
fc = 1200                    # Hz — à adapter selon le signal
fcn = fc / fe                 # fréquence normalisée [0, 0.5]
delta_f = fe / 100           # bande de transition (Δf ≤ fe/100)

# Hamming → K = 6.6
N_ordre = int(np.ceil(6.6 * fe / delta_f))
if N_ordre % 2 == 0: N_ordre += 1   # N impair obligatoire

b = sp.firwin(N_ordre, fcn, window='hamming', pass_zero='lowpass')
a = [1]

# ==== FILTRAGE ====
y = sp.lfilter(b, a, x_bruite)

Résultat attendu : Après filtrage, les pics spectraux à 1500, 2000 et 3000 Hz disparaissent. Le signal filtré est quasi identique au signal original. Le délai de groupe = (N−1)/2 échantillons.

TP 03

Synthèse de Filtres RII — Méthode Bilinéaire TP3

1. Filtre RII vs RIF

Filtre RIF
  • A(z) = 1, uniquement des zéros
  • Toujours stable (pas de pôles)
  • Phase linéaire possible
  • Ordre élevé (centaines de coeff.)
  • Méthode : fenêtres, firwin()
Filtre RII
  • A(z) ≠ 1, pôles et zéros
  • Stable si |pôles| < 1
  • Phase non linéaire en général
  • Ordre faible (ex: ordre 3–8)
  • Méthode : transformation bilinéaire

2. Transformation bilinéaire

p = (2/Te) · (1 − z⁻¹) / (1 + z⁻¹) = 2·Fe · (1 − z⁻¹) / (1 + z⁻¹)

Cette substitution convertit un filtre analogique H(p) en filtre numérique H(z). Elle préserve la stabilité et les bandes passantes/atténuées.

Pipeline de synthèse RII
Specs numériques fp,fa,Ap,Aa → Specs analogiques fpa,faa → Filtre prototype → Dénormalisation → H(z) par p→z
fp_analogique = Fe · tan(π·fp/Fe)    |    fa_analogique = Fe · tan(π·fa/Fe)

3. Les 3 filtres — Code Python

# Spécifications communes
Aa, Ap = 50, 2             # dB
delta_f = fe / 50
fc = 1200                  # fréquence de coupure (Hz)
fp = fc - delta_f/2
fa = fc + delta_f/2

# ---- Butterworth ----
N_but, fc_but = sp.buttord(fp, fa, Ap, Aa, analog=False, fs=fe)
b_but, a_but = sp.butter(N_but, fc_but, 'low', False, 'ba', fe)

# ---- Chebyshev type 1 ----
N_c1, fc_c1 = sp.cheb1ord(fp, fa, Ap, Aa, analog=False, fs=fe)
b_c1, a_c1 = sp.cheby1(N_c1, Ap, fc_c1, 'low', False, 'ba', fs=fe)

# ---- Chebyshev type 2 ----
N_c2, fc_c2 = sp.cheb2ord(fp, fa, Ap, Aa, analog=False, fs=fe)
b_c2, a_c2 = sp.cheby2(N_c2, Aa, fc_c2, 'low', False, 'ba', fs=fe)

# ---- Vérification stabilité ----
poles = sp.tf2zpk(b_but, a_but)[1]
stable = np.all(np.abs(poles) < 1)
print(f"Stable: {stable}")

4. Comparaison des filtres analogiques

Filtre Bande passante Bande atténuée Ordre N typique Avantage
Butterworth Monotone (pas d'ondulations) Monotone Le plus élevé Réponse la plus plate en BP
Chebyshev 1 Ondulations (ripple) ≤ Ap Monotone Moyen Transition plus raide que Butter.
Chebyshev 2 Monotone Ondulations ≤ Aa Moyen BP propre, atténuation garantie
Elliptique Ondulations Ondulations Le plus faible Ordre minimal pour specs données

Réponse à "lequel est le plus adéquat ?" : Le filtre elliptique (Cauer) donne l'ordre le plus faible pour des spécifications données, donc moins de calculs. Si la phase est critique → Butterworth. Pour un compromis ordre/ondulations → Chebyshev 2 (BP propre).

TP 04

Filtrage Multicadence TP4

1. Sous-échantillonnage (Décimation)

Théorème de Shannon : Pour éviter le repliement (aliasing), il faut que fe ≥ 2·fmax. Si on décime par D, la nouvelle fe = fe/D doit rester ≥ 2·fmax.

Sans filtrage (direct)

D = 2
x_dec = x[::2]     # prendre 1 sur 2
fe_new = fe // D

Produit du repliement spectral (aliasing) si D trop grand.

Avec filtrage (sp.decimate)

x_dec = sp.decimate(x, D)

Applique un filtre anti-repliement Chebyshev ordre 8 (par défaut) avant de sous-échantillonner. Pas d'aliasing.

2. Sur-échantillonnage (Interpolation)

Insertion de zéros (sans filtre)

D = 4
x_up = np.zeros(D*len(x))
x_up[::4] = x
fe_up = fe * D

Crée des images spectrales. Le signal a l'air correct mais le spectre contient des copies non désirées.

Avec filtrage (sp.resample)

x_up = sp.resample(x, D*len(x))

Interpolation propre par transformée de Fourier + filtrage passe-bas. Élimine les images spectrales.

3. Changement de fréquence rationnel (L/M)

Pour passer de fe à 1.5·fe (facteur 3/2) :

# Solution 1 : Sur-échantillonner par 3, sous-échantillonner par 2
x_rat1 = sp.resample(x, 3*len(x))[::2]

# Solution 2 : resample direct (meilleure — conserve les valeurs originales)
x_rat2 = sp.resample(x, int(len(x) * 1.5))

4. Résumé des effets à l'écoute

Actionfe utilisée pour lectureEffet entendu
Décimation D=2, lecture à fe fe original (trop rapide) Son accéléré, grave coupé
Décimation D=2, lecture à fe/2 fe/2 (correct) Durée normale, aliasing possible si sans filtre
Sur-éch. D=4, insertion zéros, lecture à 4·fe 4·fe Son correct mais bruit d'images spectrales
Sur-éch. D=4, sp.resample, lecture à 4·fe 4·fe Son propre, qualité préservée
💡

Jusqu'à quel D peut-on décimer sans déformation ? Tant que fe_new = fe/D ≥ 2·fmax du signal. Si fmax ≈ 4000 Hz et fe = 22050 Hz → D_max = floor(22050 / (2×4000)) ≈ 2. Au-delà, repliement audible.

COURS

Questions de Cours Essentielles

Théorème de Shannon-Nyquist

Énoncé : Pour reconstruire parfaitement un signal analogique depuis ses échantillons, la fréquence d'échantillonnage doit satisfaire :

fe ≥ 2 · fmax

Si cette condition n'est pas respectée → repliement spectral (aliasing) : des fréquences hautes semblent être des fréquences basses, le signal est irrémédiablement corrompu.

Stabilité d'un filtre numérique

Un filtre RII est stable si et seulement si tous ses pôles sont à l'intérieur du cercle unité :

|pᵢ| < 1 pour tout pôle pᵢ

En Python : np.all(np.abs(sp.tf2zpk(b,a)[1]) < 1)

Un filtre RIF (A(z)=1) est toujours stable car il n'a pas de pôles.

Réponse impulsionnelle vs fréquentielle

Réponse impulsionnelle h(n)

Réponse du filtre à une impulsion de Dirac δ(n). C'est la transformée en z inverse de H(z). Pour un filtre RIF, h(n) = b[n] directement.

Réponse fréquentielle H(f)

Gain du filtre à chaque fréquence. Obtenu en évaluant H(z) sur le cercle unité (z = e^(j2πf/fe)). En Python : sp.freqz(b, a)

Diagramme pôles-zéros

  • Zéro en z₀ → H(z₀) = 0 → atténuation totale à la fréquence f₀ = Fe/(2π)·angle(z₀)
  • Zéro en z = +1 (angle 0) → atténuation à f = 0 Hz (passe-haut)
  • Zéro en z = −1 (angle π) → atténuation à f = fe/2 (passe-bas)
  • Pôle proche du cercle unité → pic de résonance à la fréquence correspondante
  • Cercle unité : stabilité (pôles intérieurs = stable)

Délai de groupe d'un filtre RIF

Un filtre RIF à phase linéaire introduit un délai constant de (N−1)/2 échantillons. C'est pourquoi le signal filtré est décalé dans le temps.

Aliasing / Repliement spectral

Si fe < 2·fmax, les fréquences au-dessus de fe/2 se "replient" et apparaissent à des fréquences plus basses : faliasée = |f − k·fe| pour l'entier k le plus proche. Ce phénomène est irréversible. Solution : filtrer passe-bas AVANT de sous-échantillonner (filtre anti-repliement).

Normalisation des fréquences

fcn = fc / fe ∈ [0, 0.5] (normalisée par rapport à fe)

Les fonctions Python comme firwin() attendent des fréquences normalisées entre 0 et 0.5 (ou 0 et 1 selon la convention). Toujours vérifier la documentation.

Exercice 1 — TFD : Analyse spectrale d'un signal composite
TP1 · Analyse fréquentielle
DIFFICILE ▼

Q1. On dispose d'un signal x(t) = 3·cos(2π·500t) + 1.5·sin(2π·1200t) + 2·cos(2π·3000t) échantillonné à fe = 8000 Hz. On calcule sa TFD sur N = 8000 points. À quels indices k apparaissent les composantes fréquentielles dans le spectre (avant fftshift) ?

Rappel : k = f·N/fe. Pour la fréquence négative : k = N − k_positif

Pour 500 Hz : k = 500 × 8000 / 8000 = 500 et k = 8000 − 500 = 7500

Pour 1200 Hz : k = 1200 × 8000 / 8000 = 1200 et k = 6800

Pour 3000 Hz : k = 3000 × 8000 / 8000 = 3000 et k = 5000

Après fftshift, ces indices se déplacent : k_centré = k − N/2. Les amplitudes spectrales sont A/2 (le signal étant réel, chaque fréquence apparaît deux fois).

Q2. Écrire le code Python qui génère ce signal, calcule sa TFD, et trace son spectre d'amplitude en dB entre −fe/2 et +fe/2. Pourquoi doit-on ajouter 1e-12 avant le log ?

import numpy as np
import matplotlib.pyplot as plt

fe, N = 8000, 8000
t = np.arange(N) / fe
x = (3*np.cos(2*np.pi*500*t) + 1.5*np.sin(2*np.pi*1200*t)
     + 2*np.cos(2*np.pi*3000*t))

TFx = np.fft.fftshift(np.fft.fft(x, N) / N)
TFxdB = 20*np.log10(np.abs(TFx) + 1e-12)
f = np.arange(-N/2, N/2) * fe / N

plt.plot(f, TFxdB); plt.grid(); plt.show()

+1e-12 : Sans ça, là où TFx = 0, on calcule log10(0) = −∞, ce qui provoque une erreur ou des valeurs invalides dans le tracé.

Q3. Si on augmente N (nombre de points FFT) sans changer le signal, qu'arrive-t-il à la résolution fréquentielle ? Est-ce que ça améliore la précision des amplitudes ?

Résolution fréquentielle : Δf = fe/N. Augmenter N améliore la résolution (Δf diminue) → les pics sont plus fins.

Amplitudes : Si on calcule N plus grand que la longueur du signal (zero-padding), la résolution augmente mais pas la précision réelle — on interpole le spectre existant. Si N = longueur réelle du signal, plus de points → meilleure précision. La normalisation par N garde les amplitudes cohérentes.

Exercice 2 — Conception d'un filtre RIF passe-bas
TP2 · Méthode des fenêtres
DIFFICILE ▼

Données : fe = 16 000 Hz, fp = 2000 Hz, fa = 2400 Hz, Aa > 50 dB

Q1. Déterminer la fenêtre appropriée, la fréquence de coupure fc, la bande de transition Δf, et l'ordre du filtre N.

Fenêtre : Aa > 50 dB → table : Hamming (Aa = 53 dB ≥ 50 dB) ✓

Δf : fa − fp = 2400 − 2000 = 400 Hz

fc : (fp + fa)/2 = (2000 + 2400)/2 = 2200 Hz

N : K × Fe/Δf = 6.6 × 16000/400 = 264 → arrondir au suivant impair = 265

fcn : fc/fe = 2200/16000 = 0.1375

Q2. Écrire le code Python pour concevoir ce filtre et filtrer un signal bruité x_b.

fe = 16000
fp, fa, Aa = 2000, 2400, 50
fc = (fp + fa) / 2      # 2200 Hz
fcn = fc / fe           # 0.1375
delta_f = fa - fp       # 400 Hz
N = int(np.ceil(6.6 * fe / delta_f))
if N % 2 == 0: N += 1   # N = 265

b = sp.firwin(N, fcn, window='hamming', pass_zero='lowpass')
a = [1]
y = sp.lfilter(b, a, x_b)

Q3. Quel est le délai introduit par ce filtre ? Comment le compenser dans le tracé ?

Délai : τ = (N−1)/2 = (265−1)/2 = 132 échantillons = 132/16000 ≈ 8.25 ms

Compensation : Pour aligner les signaux dans le tracé : y_aligne = y[(N-1)//2:] puis tronquer x_b au même nombre de points.

Q4. Si on veut Aa > 70 dB avec la même bande de transition, quelle fenêtre choisir ? Quel est le nouvel ordre ?

Fenêtre : Aa > 70 dB → Blackman (Aa = 74 dB) ✓

N : K = 11 → N = ceil(11 × 16000/400) = ceil(440) = 441 (déjà impair)

L'ordre est presque doublé par rapport à Hamming (441 vs 265), ce qui signifie plus de calculs et plus de mémoire.

Exercice 3 — Filtre RII par transformation bilinéaire
TP3 · Butterworth, Chebyshev, stabilité
EXPERT ▼

Données : fe = 44100 Hz, Aa = 60 dB, Ap = 1 dB, Δf ≤ fe/50

Q1. Calculer la bande de transition maximale Δf et définir fp et fa pour une fréquence de coupure fc = 3000 Hz.

Δf_max : fe/50 = 44100/50 = 882 Hz

On choisit Δf = 882 Hz (valeur limite)

fp : fc − Δf/2 = 3000 − 441 = 2559 Hz

fa : fc + Δf/2 = 3000 + 441 = 3441 Hz

Q2. Comparer les ordres N obtenus avec Butterworth, Chebyshev 1 et Chebyshev 2 pour ces spécifications. Quel filtre vous choisissez et pourquoi ?

fe = 44100; Aa, Ap = 60, 1
fp, fa = 2559, 3441

N_but, _ = sp.buttord(fp, fa, Ap, Aa, fs=fe)    # typiquement ≈ 10–14
N_c1,  _ = sp.cheb1ord(fp, fa, Ap, Aa, fs=fe)   # typiquement ≈ 6–9
N_c2,  _ = sp.cheb2ord(fp, fa, Ap, Aa, fs=fe)   # typiquement ≈ 6–9

Choix : Chebyshev 2 si la bande passante doit être lisse (pas d'ondulations). Butterworth si la phase est importante. Pour un usage audio, Chebyshev 2 est souvent le meilleur compromis.

Q3. Comment vérifier la stabilité d'un filtre RII numériquement ? Donner le code Python et la condition mathématique précise.

Condition : Tous les pôles pᵢ doivent satisfaire |pᵢ| < 1

_, poles, _ = sp.tf2zpk(b, a)
stable = np.all(np.abs(poles) < 1)
print(f"Pôles: {poles}")
print(f"Modules: {np.abs(poles)}")
print(f"Stable: {stable}")

Les fonctions scipy (butter, cheby1, cheby2) garantissent des filtres stables par construction.

Q4 (piège d'examen). Quelle est la différence fondamentale entre sp.freqz(b,a) et sp.freqz(b,a,fs=fe) ? Quel axe des fréquences obtient-on dans chaque cas ?

Sans fs : Les fréquences sont en radians/échantillon, de 0 à π. L'axe renvoie ω ∈ [0, π].

Avec fs=fe : Les fréquences sont en Hz, de 0 à fe/2. L'axe renvoie f ∈ [0, fe/2].

Pour un tracé en Hz correct : toujours passer fs=fe. Sinon les labels de l'axe sont faux.

Exercice 4 — Multicadence : décimation et interpolation
TP4 · Shannon, Aliasing, Resample
EXPERT ▼

Q1. Un signal x(t) = cos(2π·1000t) + cos(2π·3500t) est échantillonné à fe = 8000 Hz. On le décime par D = 3. À quelles fréquences apparaissent les composantes dans le signal décimé (sans filtre) ?

Rappel : faliasée = |f − round(f/fe_new) × fe_new| avec fe_new = fe/D

fe_new = 8000/3 ≈ 2667 Hz

Composante à 1000 Hz : 1000 < 2667/2 = 1333 Hz → pas de repliement → reste à 1000 Hz

Composante à 3500 Hz : 3500 > 1333 Hz → repliement ! faliasée = |3500 − round(3500/2667) × 2667| = |3500 − 1 × 2667| = 833 Hz

La composante à 3500 Hz apparaît à 833 Hz → elle se superpose au signal utile → impossible à séparer !

Q2. Que fait exactement sp.decimate(x, D) ? Quel filtre utilise-t-il ? Quelle est sa fréquence de coupure ?

Opération : sp.decimate(x, D) effectue deux opérations en séquence :

1. Applique un filtre anti-repliement passe-bas (filtre de Chebyshev de type 1, ordre 8 par défaut)

2. Sous-échantillonne par D (garde 1 échantillon sur D)

Fréquence de coupure du filtre : fc = fe/(2·D) — exactement à la moitié de la nouvelle fréquence de Nyquist.

Q3. On veut passer d'une fréquence fe = 44100 Hz à fe_new = 32000 Hz (facteur rationnel). Proposer deux méthodes et déterminer les facteurs L et M.

Facteur : fe_new/fe = 32000/44100 = 320/441

Méthode 1 (sur-éch. puis décim.) : Sur-échantillonner par L=320, puis décimer par M=441.

x_up = sp.resample(x, 320*len(x))
x_new = x_up[::441]

Méthode 2 (directe) :

x_new = sp.resample(x, int(len(x)*32000/44100))

La méthode directe est meilleure (une seule opération, meilleure qualité).

Q4 (piège d'examen). Si on sur-échantillonne par 4 en insérant des zéros puis on lit à 4·fe, pourquoi entend-on quand même le son correct malgré les zéros insérés ?

L'insertion de zéros ne change pas le contenu fréquentiel fondamental du signal — les zéros ne sont pas du "silence" mais des interpolations nulles entre les échantillons. Le système de lecture interprète tous les échantillons selon fe_new = 4·fe, ce qui revient à "étaler" le signal dans le temps (durée×4). Le son original est contenu dans la bande [0, fe/2], qui est toujours présente dans [0, 4·fe/2] = [0, 2·fe]. Cependant, des images spectrales apparaissent aux multiples de fe, ce qui crée un son "métallique". Le filtre passe-bas (sp.resample) élimine ces images.

Exercice 5 — Questions de cours difficiles
Théorie · Tous TPs

Q1. Expliquer pourquoi fftshift est nécessaire et ce qu'il fait exactement sur le résultat de np.fft.fft().

np.fft.fft() retourne les fréquences dans l'ordre : [0, Δf, 2Δf, ..., fe/2, −fe/2+Δf, ..., −Δf]. Les fréquences négatives sont dans la deuxième moitié du tableau.

fftshift réarrange le tableau pour obtenir : [−fe/2, ..., −Δf, 0, Δf, ..., fe/2]. Cela permet de tracer le spectre bilatéral centré autour de 0 Hz, ce qui est la représentation physiquement intuitive.

Q2. Un filtre RIF a les coefficients b = [1, 0, −1]. Déterminer sa réponse en fréquence H(f) analytiquement. Quel type de filtre est-ce ?

H(z) = 1 + 0·z⁻¹ − 1·z⁻² = 1 − z⁻²

En z = e^(j2πf/fe) : H(f) = 1 − e^(−j4πf/fe)

|H(f)| = |1 − e^(−j4πf/fe)| = 2|sin(2πf/fe)|

À f = 0 : |H(0)| = 0 → atténuation totale. À f = fe/4 : |H| = 2 → gain max. → C'est un filtre passe-haut différentiateur (type dérivateur numérique).

Q3. Quelle est la différence entre sp.lfilter(b,a,x) et sp.filtfilt(b,a,x) ? Quel est l'avantage/inconvénient de chacun ?

lfilter : Filtrage causal (dans le sens chronologique). Introduit un délai de groupe = (N−1)/2. Signal filtré décalé par rapport à l'original.

filtfilt : Filtrage avant ET arrière (aller-retour). Délai nul (phase nulle). Atténuation doublée (ordre effectif doublé). Ne peut pas être utilisé en temps réel car il nécessite le signal complet.

Pour l'examen TNS : On utilise lfilter (traitement hors-ligne des TPs). filtfilt serait utilisé pour des applications où la phase importe.

Q4. Lors du formatage d'un signal flottant y en uint8, expliquer ligne par ligne la formule : yf = np.uint8((y-min(y))*(255)/(max(y)-min(y)))

y − min(y) : Décale le signal pour que sa valeur minimale soit 0 (signal positif)

/ (max(y) − min(y)) : Normalise sur [0, 1]

× 255 : Met à l'échelle sur [0, 255] (255 niveaux d'un uint8)

np.uint8(...) : Convertit en entiers non signés 8 bits (troncature vers le bas)

Pour int16, on soustrait 2¹⁵ = 32768 après la mise à l'échelle pour centrer le signal autour de 0.

Q5 (très difficile). Un signal audio de 3 secondes échantillonné à 22050 Hz est filtré par un filtre RIF d'ordre 660. Quel est le retard introduit ? Le signal filtré a-t-il la même durée ? Comment obtenir la même durée sur les deux signaux pour comparer ?

Retard : τ = (N−1)/2 = (661−1)/2 = 330 échantillons = 330/22050 ≈ 15 ms

Durée du signal filtré : sp.lfilter retourne exactement N_x échantillons (même longueur que l'entrée), mais les 330 derniers échantillons correspondent à des "effets de bord" (extinction du filtre).

Alignement :

y_aligne = y[330:]          # décaler y de 330 samples
x_court = x[:len(y_aligne)] # tronquer x à la même longueur

Ainsi x_court et y_aligne ont la même durée et sont temporellement alignés.

Exercice 6 — Problème de synthèse complet (type examen final)

Énoncé : On a un signal vocal échantillonné à fe = 44100 Hz. Des interférences à 5000, 8000 et 12000 Hz d'amplitude 100V corrompent le signal. On veut concevoir un système complet de débruitage.

Partie A. Générer le signal bruité et tracer son spectre. Indiquer à quels indices k les bruits apparaissent dans le spectre unilatéral (0 à fe/2).

t = np.arange(len(x)) / fe
bruit = 100 * (np.sin(2*np.pi*5000*t) +
               np.sin(2*np.pi*8000*t) +
               np.sin(2*np.pi*12000*t))
x_b = x.astype(float) + bruit

Indices k dans le spectre (N points) : k = f·N/fe

5000 Hz → k = 5000·N/44100 ≈ 0.113·N

8000 Hz → k ≈ 0.181·N ; 12000 Hz → k ≈ 0.272·N

Partie B. On veut utiliser un filtre RIF passe-bas avec fc = 4000 Hz, Δf = 500 Hz, Aa > 50 dB. Calculer l'ordre et écrire le code de conception.

Aa > 50 dB → Hamming (53 dB ≥ 50 dB)

N = ceil(6.6 × 44100 / 500) = ceil(583.2) = 584 → impair : N = 585

fcn = 4000/44100 = 0.0907

b = sp.firwin(585, 0.0907, window='hamming', pass_zero='lowpass')
y_rif = sp.lfilter(b, [1], x_b)

Partie C. On décide d'utiliser un filtre RII Butterworth à la place. Avec Ap = 1 dB, Aa = 55 dB, fp = 3750 Hz, fa = 4250 Hz. Écrire le code et comparer visuellement les deux solutions.

Ap, Aa = 1, 55
fp, fa = 3750, 4250
N_b, fc_b = sp.buttord(fp, fa, Ap, Aa, fs=fe)
b_b, a_b = sp.butter(N_b, fc_b, 'low', fs=fe)
y_rii = sp.lfilter(b_b, a_b, x_b)

print(f"Ordre Butterworth: {N_b}")
print(f"Ordre RIF Hamming: 585")

Comparaison attendue : Butterworth N ≈ 8 (vs 585 pour RIF !). Mais le filtre RIF a une phase linéaire (délai constant), le RII a une phase non-linéaire. Sur un signal audio, le RII peut introduire une distorsion de phase perceptible.

Partie D. On veut compresser le fichier en le rééchantillonnant à 22050 Hz (D=2). Expliquer la procédure complète (avec filtre anti-repliement) et écrire le code.

Procédure :

1. Vérifier que le contenu utile est < 22050/2 = 11025 Hz (ici oui, le bruit est déjà filtré)

2. Appliquer sp.decimate qui filtre anti-repliement + sous-échantillonne

y_filtre = sp.lfilter(b, [1], x_b)   # d'abord filtrer le signal
y_dec = sp.decimate(y_filtre, 2)          # puis décimer
fe_new = fe // 2                          # = 22050 Hz

# Formatage pour sauvegarde
y16 = np.int16((y_dec - y_dec.min()) / (y_dec.max() - y_dec.min()) * 65535 - 32768)
wav.write('sortie_decime.wav', fe_new, y16)

Pièges classiques d'examen

  • firwin attend fcn ∈ [0, 0.5], PAS en Hz !
  • N doit être impair pour un filtre RIF symétrique à phase linéaire
  • Toujours ajouter 1e-12 avant le log10
  • Diviser la FFT par N pour normaliser les amplitudes
  • fftshift est nécessaire pour un spectre centré sur 0 Hz
  • Convertir x en float64 avant d'ajouter le bruit pour éviter le dépassement int
  • La stabilité se vérifie avec |pôles| < 1, pas <= 1
  • sp.decimate utilise Chebyshev ordre 8 par défaut, coupure à fe/(2D)
