Formules clés à connaître

H = Σ P(si) × log₂(1/P(si)) bits/symbole
Taux de compression = (N × bpp_original) / (N_codes × 32)
Débit = (N_codes × 32) / N bits/échantillon
MSE = mean( (original - reconstruit)² )
PSNR = 20 × log₁₀(255 / √MSE) dB

Calcul des symboles et probabilités — Compt_symb_prob(L)

Cette fonction est commune aux 3 types (texte, audio, image). Elle construit une liste de dictionnaires [{'Symb': x, 'Prob': p}, …]

def Compt_symb_prob(L):
    N = len(L)          # Nombre total d'éléments
    Symb = []
    Symb.append(L[0])   # Premier symbole distinct
    F = np.zeros(N)    # Tableau de fréquences
    F[0] = 1
    k = 1               # Compteur de symboles distincts
    for i in range(1,N):
        u = 0
        # Cherche si L[i] est déjà dans Symb
        while (u < k) and (L[i]!=Symb[u]):
            u=u+1
        if (u==k):        # Nouveau symbole
            Symb.append(L[i])
            F[k] = 1
            k=k+1
        else:              # Symbole existant → incrémenter
            F[u]=F[u]+1
    Symb_Prob=[]
    for i in range(k):
        D={}
        D['Symb']=Symb[i]
        D['Prob']=F[i]/N  # Probabilité = fréquence/total
        Symb_Prob.append(D)
    return Symb_Prob

📌 Logique : Pour chaque élément L[i], on parcourt la liste des symboles déjà trouvés. Si u atteint k (fin de liste sans trouver) → nouveau symbole. Sinon → incrémenter le compteur du symbole trouvé à l'indice u.

Calcul de l'entropie

H = 0
for i in range(Nb):
    H = H + Symb_Prob[i]['Prob'] * np.log2(1/Symb_Prob[i]['Prob'])
print(f"Entropie={H}")

Interprétation de l'entropie

Entropie HSignificationCompressibilité
H ≈ 0Source très prévisible (quasi-un symbole)Très bonne
H élevé (proche log₂(Nb))Symboles équiprobables, aléatoireTrès faible
H ≈ log₂(Nb)Distribution uniforme, entropie maximaleNulle

✅ Règle : Le débit du codage entropique ne peut pas descendre sous H (borne théorique). Huffman et Arithmétique tendent vers H.

TP2 – Codage de Huffman

Principe

Attribuer des codes binaires courts aux symboles fréquents, longs aux rares. Code préfixe (aucun code n'est le préfixe d'un autre).

  1. Calculer les fréquences de chaque symbole
  2. Construire un arbre binaire (tas min) : fusionner les deux noeuds de plus faible fréquence
  3. Générer les codes : gauche = 0, droite = 1
  4. Encoder : remplacer chaque symbole par son code
  5. Décoder : parcourir l'arbre bit par bit

Classe Node et construction de l'arbre

class Node:
    def __init__(self, char, freq):
        self.char = char
        self.freq = freq
        self.left = None
        self.right = None

    def __lt__(self, other):   # Permet la comparaison dans heapq
        return self.freq < other.freq

def build_huffman_tree(valeurs):
    frequency = Compt_symb_freq(valeurs)
    heap = []
    for elem in frequency:
        heap.append(Node(elem['char'], elem['Freq']))
    heapq.heapify(heap)          # Transforme en tas min
    while len(heap) > 1:
        left  = heapq.heappop(heap)  # Freq la plus faible
        right = heapq.heappop(heap)  # 2e freq la plus faible
        merged = Node(None, left.freq + right.freq)
        merged.left = left
        merged.right = right
        heapq.heappush(heap, merged)
    return heap[0]                 # Racine de l'arbre

Génération des codes & Encodage / Décodage

def build_huffman_codes(root):
    codes = {}
    def generate_codes(node, current_code):
        if node is None: return
        if node.char is not None:   # Feuille = symbole
            codes[node.char] = current_code
        generate_codes(node.left,  current_code+"0")
        generate_codes(node.right, current_code+"1")
    generate_codes(root, "")
    return codes

def huffman_encode(text, codes):
    return ''.join(codes[char] for char in text)

def huffman_decode(encoded_text, root):
    decoded = []
    node = root
    for bit in encoded_text:
        if bit == '0': node = node.left
        else:          node = node.right
        if node.char is not None:  # Feuille atteinte
            decoded.append(node.char)
            node = root           # Remonter à la racine
    return ''.join(decoded)

📌 Décodage : On parcourt l'arbre bit par bit. Quand on atteint une feuille (noeud.char ≠ None), on a trouvé un symbole, on repart de la racine.

Taux de compression et débit

print(f"Taux={len(valeurs)*8/len(encoded_text)}")
# len(valeurs)*8 = taille originale en bits (8 bits/octet)

TP3 – Codage Arithmétique

Principe général

Au lieu de coder symbole par symbole, on encode un paquet entier par un nombre réel dans [0,1[. Chaque symbole raffine l'intervalle.

Étape 1 — Créer les intervalles de symboles
def create_Symb_Interval(Symb_Prob):
    Nb=len(Symb_Prob)
    IS=np.zeros(Nb*2)
    IS.shape=(Nb,2)   # Matrice Nb×2 : [borne_inf, borne_sup]
    IS[0,0]=0
    IS[0,1]=Symb_Prob[0]['Prob']
    for i in range(1,Nb):
        IS[i,0]=IS[i-1,1]           # Borne inf = borne sup du précédent
        IS[i,1]=IS[i,0]+Symb_Prob[i]['Prob']
    return IS  # Intervalles contigus qui couvrent [0,1]

📌 Résultat : chaque symbole sᵢ est associé à [ISᵢ,₀ ; ISᵢ,₁[. La largeur de l'intervalle = P(sᵢ).

Étape 2 — Encodage par paquets
def Arithm_encode(L, Symb_Prob, IS, nomf_comp):
    Long=int(input("Longueur du paquet :"))
    N=len(L)
    r=N%Long    # Reste (dernier paquet incomplet)
    Np=N//Long  # Nombre de paquets complets

    for i in range(Np):
        Paquet=L[i*Long:(i+1)*Long]
        Iinf=0; Isup=0; Largeur=1
        for j in range(Long):
            u=0
            # Trouver quel symbole = Paquet[j]
            while(u<Nb) and (Paquet[j]!=Symb_Prob[u]['Symb']):
                u=u+1
            # Réduire l'intervalle
            Isup=Iinf+Largeur*IS[u,1]
            Iinf=Iinf+Largeur*IS[u,0]
            Largeur=Isup-Iinf
        Code[i]=(Iinf+Isup)/2  # Milieu de l'intervalle final

⚠️ Ligne clé examen : Isup=Iinf+Largeur*IS[u,1] et Iinf=Iinf+Largeur*IS[u,0] → réduction de l'intervalle courant au sous-intervalle du symbole u. La largeur diminue à chaque symbole.

Étape 3 — Décodage
for j in range(Long):
    u=0
    # Trouver dans quel intervalle tombe Code[i]
    while (u<Nb) and (Code[i]<IS[u,0] or Code[i]>IS[u,1]):
        u=u+1
    Donnees.append(Symb_Prob[u]['Symb'])
    # Mise à jour du code (normalisation)
    Code[i]=(Code[i]-IS[u,0])/(IS[u,1]-IS[u,0])

📌 Mise à jour du code : On ramène Code[i] dans [0,1] en soustrayant la borne inf du symbole trouvé et en divisant par sa largeur d'intervalle. C'est l'opération inverse de l'encodage.

Questions clés du TP3

Q1 — Appartenance à un intervalle

Lignes de décodage : le while cherche u tel que IS[u,0] ≤ Code[i] ≤ IS[u,1]. Le symbole u est alors l'élément décodé.

Q2 — Mise à jour du code

Code[i] = (Code[i] - IS[u,0]) / (IS[u,1] - IS[u,0]) — normalisation dans [0,1] pour décoder le symbole suivant.

Q3 — Vérification exacte (texte)

Comparaison directe : if Chaine == chaine1: print("Réussie")

Q4 — Vérification approchée (audio/image)

Audio : MSE < 10⁻¹ → acceptable. Image : PSNR en dB (plus élevé = meilleur). Arithmétique sur float → erreurs d'arrondi inévitables.

Q5 — Pourquoi les erreurs augmentent avec Long ?

Plus le paquet est long, plus la largeur de l'intervalle diminue exponentiellement (Largeur → 0). La précision des flottants Python (64 bits) ne suffit plus → erreurs de décodage à partir d'une certaine longueur.

TP4 – Canal bruité & Codage correcteur (MATLAB)

Objectif

Comparer les performances (BER) de transmissions avec et sans codage correcteur d'erreur dans un canal AWGN (bruit gaussien blanc additif).

Concepts clés

BER — Bit Error Rate

Taux d'erreur binaire = nombre de bits erronés / nombre total de bits transmis.

BER = erreurs / N_bits
Canal AWGN

Additive White Gaussian Noise. Le bruit ajouté suit une loi gaussienne. Paramètre : Eb/N0 (rapport signal sur bruit en dB).

Structure du programme de simulation

  1. Générer N=10⁶ bits aléatoires : x = randi([0,1], 1, Nb)
  2. Encoder avec le code choisi (linéaire, Hamming, cyclique, convolutif)
  3. Passer dans le canal bruité : y = awgn(signal, snr, 'measured')
  4. Décider (seuil 0) puis calculer le BER
  5. Décoder avec la fonction correspondante
  6. Tracer BER vs Eb/N0 (échelle semilog)

Ligne 4 : Le code du paquet = milieu de l'intervalle final. On peut décoder tous les symboles du paquet à partir de ce seul nombre.

Question : Pourquoi gère-t-on le cas r != 0 séparément ? Qu'est-ce que r ?

r = N % Long est le reste de la division de N (taille totale) par Long (taille d'un paquet). Si r ≠ 0, il existe un dernier paquet incomplet de r symboles. Ce cas doit être traité séparément car la boucle principale traite uniquement des paquets complets de taille Long. Le dernier paquet de taille r est extrait avec L[i*Long : i*Long+r] et Np est incrémenté pour compter ce paquet supplémentaire.
