EXERCICE 1 · 7 pts

Instructions Thumb : MOV, ADD, SUBS, BNE, STR

Comprendre et appliquer les instructions ARM Thumb de base pour manipuler les registres, la mémoire et les flags.

📋

PART A — Questions de compréhension (2,5 pts)

Q1 · 0.5ptQue fait MOV R0, #5 ?▶
Réponse : Cette instruction charge la constante immédiate 5 dans le registre R0. C'est une opération de chargement direct (pas de lecture mémoire). Après l'instruction : R0 = 5. Les flags NZCV ne sont PAS modifiés (pour mettre à jour les flags il faudrait MOVS).
Syntaxe Thumb : MOV Rd, #imm8 — l'immédiat est sur 8 bits (0–255).
Q2 · 0.5ptDifférence entre ADD R1, R0, R2 et ADDS R1, R0, R2 ?▶
ADDADDS
R1 ← R0 + R2R1 ← R0 + R2
Flags NZCV non modifiésFlags NZCV mis à jour
Pas de branchement conditionnel possible aprèsBNE / BEQ / BCS... utilisables après
Règle clé : Le suffixe S (ADDS, SUBS, MOVS...) signifie «Set flags». Sans S → les flags ne changent pas.
Q3 · 0.5ptEffet de SUBS R3, R3, #1 sur les flags N, Z, C, V▶
FlagSignificationMis à 1 si...
N — NegativeRésultat négatifBit 31 du résultat = 1 (R3 devient négatif)
Z — ZeroRésultat nulR3 − 1 = 0 ← condition pour arrêter une boucle
C — CarryDépassement non signéEmprunt lors de la soustraction (borrow)
V — OverflowDépassement signéDébordement arithmétique signé
Exemple : Si R3 = 1, après SUBS R3, R3, #1 → R3 = 0, Z = 1, N = 0, C = 1 (pas d'emprunt), V = 0.
Q4 · 0.5ptSous quelle condition BNE label branche-t-il ?▶
BNE = Branch if Not Equal = Branch if Zero flag = 0
BNE branche vers label si le flag Z = 0 (résultat de la dernière opération ≠ 0).
Si Z = 1 (résultat = 0), le branchement ne se fait pas et l'exécution continue normalement.

C'est la clé des boucles de décompte : tant que SUBS n'arrive pas à 0, Z=0 et BNE boucle.

Q5 · 0.5ptRôle de STR R2, [R4]▶
STR = Store Register — stocke le contenu d'un registre en mémoire.
STR R2, [R4] : écrit la valeur de R2 à l'adresse mémoire pointée par R4.
R4 est utilisé comme pointeur. La mémoire à l'adresse R4 reçoit la valeur de R2. R4 lui-même n'est pas modifié.
InstructionSensMémoire ↔ Registre
STR Rt, [Rn]Registre → MémoireÉcriture
LDR Rt, [Rn]Mémoire → RegistreLecture

Q9 · 0.5ptComment les valeurs se déplacent-elles entre registres et mémoire ?▶

🔼 Load (LDR) : Mémoire → Registre

Lit une valeur depuis l'adresse mémoire pointée par un registre, et la place dans un registre de destination.

LDR R0, [R4] → R0 ← Mémoire[R4]

🔽 Store (STR) : Registre → Mémoire

Écrit la valeur d'un registre dans la mémoire à l'adresse pointée par un autre registre.

STR R2, [R4] → Mémoire[R4] ← R2

Architecture Load/Store : Dans ARM Thumb, seules les instructions LDR et STR accèdent à la mémoire. Toutes les autres opérations (ADD, SUBS...) travaillent uniquement sur des registres.

Q4 · 0.5ptPourquoi une résistance pull-up est-elle nécessaire sur la ligne DATA ?▶
Le DHT22 utilise un protocole open-drain : la ligne DATA peut être tirée vers le bas (GND) par le capteur, mais personne ne la tire vers le haut (VCC) par défaut. La résistance pull-up (10kΩ entre DATA et VCC) assure que la ligne est à l'état HAUT au repos. Sans elle, la ligne flotterait à une tension indéfinie → données corrompues ou pas de communication.

Q6 · 0.5ptPourquoi un délai est-il nécessaire entre deux lectures DHT22 ?▶
Contrainte matérielle : Le DHT22 a un temps de conversion minimum de 2 secondes entre deux mesures. Si on lit plus vite, le capteur retourne des données invalides ou la même valeur que précédemment. La datasheet spécifie un intervalle minimum de 2s.
Solution : delay(2000); dans le loop() Arduino.
Q7 · 1ptProgramme C++ (Arduino) complet▶
ARDUINO C++
// Bibliothèque DHT pour Arduino
#include <DHT.h>

// Configuration
#define DHTPIN 2 // Broche DATA sur pin D2
#define DHTTYPE DHT22 // Type de capteur

DHT dht(DHTPIN, DHTTYPE); // Création de l'objet DHT

void setup() {
  Serial.begin(9600); // Démarrage moniteur série
  dht.begin(); // Initialisation DHT22
}

void loop() {
  delay(2000); // Attente 2s (contrainte DHT22)

  float h = dht.readHumidity(); // Lecture humidité

  // Vérification erreur lecture
  if (isnan(h) || isnan(t)) {
    Serial.println("Erreur lecture DHT22!");
    return;
  }

  // Affichage sur moniteur série
  Serial.print("Température: ");
  Serial.print(t);
  Serial.print(" °C | Humidité: ");
  Serial.print(h);
  Serial.println(" %");
}

Instruction LSL — Logical Shift Left

Comprendre et appliquer l'instruction de décalage logique à gauche avec suivi des flags.

Q9 · 0.5ptExpliquer l'opération LSL (Logical Shift Left)▶
LSL Rd, Rn, #imm : Décale tous les bits de Rn de imm positions vers la gauche. Les bits entrants à droite sont mis à 0. Le dernier bit sorti (à gauche) est copié dans le flag C (Carry).
AspectDétail
Registre RdReçoit le résultat décalé
Bits entrantsToujours 0 (côté droit)
Bit sortantCopié dans flag C
Flag Z1 si résultat = 0
Flag N1 si bit 31 du résultat = 1
Effet mathématiqueLSL #1 = ×2 ; LSL #2 = ×4
VISUALISATION LSL #1
Avant : 0000 0101
bit 7→C | ←0 entre

Après : 0000 1010
C=0 (bit sorti) · Z=0 · N=0
Q1-5 · 2.5ptsApplication complète : R0 = 0x05, LSL #1 puis #2▶
Initialisation : R0 = 0x05 = 0000 0000 0000 0000 0000 0000 0000 0101 (binaire 32 bits)
Étape Instruction Binaire (8 bits) Hex Décimal N Z C
Init MOV R0, #5 0000 0101 0x05 5 00—
Étape 1 LSL R0, R0, #1 0000 1010 0x0A 10 (×2) 000
Étape 2 LSL R0, R0, #2 0010 1000 0x28 40 (×4) 000
LSL R0, #1 :
Avant : 0000 0101 = 5
Décalage gauche 1 bit → 0 entre à droite
Après : 0000 1010 = 10
C = 0 (bit sorti = 0), Z = 0, N = 0
LSL R0, #2 :
Avant : 0000 1010 = 10
Décalage gauche 2 bits → 00 entrent à droite
Après : 0010 1000 = 40
C = 0 (dernier bit sorti = 0), Z = 0, N = 0
Règle mathématique : LSL #n est équivalent à une multiplication par 2ⁿ (tant qu'il n'y a pas de débordement). LSL #1 = ×2 ; LSL #2 = ×4 ; LSL #3 = ×8.

Cas où C = 1 (bit sort à gauche)

; Exemple avec C = 1
MOV R0, #0x80 ; R0 = 1000 0000
LSLS R0, R0, #1 ; Décale gauche de 1
; Résultat : R0 = 0000 0000 (en 8 bits)
; C = 1 (bit 7 sorti) · Z = 1 (résultat nul) · N = 0

COURS · ARCHITECTURE

Von Neumann vs Harvard — Arduino & mémoire

🧠

Von Neumann

  • Mémoire unique pour données + instructions
  • Un seul bus → traitement séquentiel
  • Goulot d'étranglement possible
  • Simple et moins cher
  • Ex : Intel 8085, Z80
🔀

Harvard

  • Mémoires séparées : données ≠ instructions
  • Bus séparés → accès parallèle
  • Plus rapide, meilleur temps réel
  • Plus complexe et coûteux
  • Ex : PIC, AVR (Arduino), ARM Cortex-M
🔵

SRAM — 2 Ko
Données temporaires (variables, pile). Volatile, perdu au reset.
EEPROM — 1 Ko
Données persistantes (configuration, calibration). Non volatile, ré-écrivable.
ARCHITECTURE INTERNE · DATAPATH ARM

Schéma interne du processeur ARM — Datapath complet

Tiré du polycopié — comprendre comment une instruction voyage depuis la mémoire jusqu'au résultat final.

🔷 Datapath ARM — vue fonctionnelle

📍

Address Register + Incrementer

Contient l'adresse mémoire courante (A31:0). L'Address Incrementer calcule automatiquement la prochaine adresse pour le fetch de l'instruction suivante. Alimenté par l'Increment Bus.

PC (R15) pointe vers la prochaine instruction. ARM mode : +4 (32 bits). Thumb mode : +2 (16 bits).
🗂️

Register Bank (R0–R15)

Banque de 16 registres + 6 registres de statut. Expose 3 ports de sortie simultanément :

  • Rm — 1ᵉʳ opérande source
  • Rn — 2ᵉ opérande source
  • Rd — Registre destination (résultat)
Lecture simultanée de 2 registres + écriture du résultat en 1 seul cycle.
⚙️

Barrel Shifter

Effectue les décalages/rotations en 1 cycle avant l'ALU. Prend l'opérande Rm et applique LSL, LSR, ASR, ROR. C'est ici que LSL R0, #1 est exécuté.

Avantage RISC : Le décalage est gratuit — intégré dans l'instruction sans cycle supplémentaire.
🔢

ALU (Arithmetic Logic Unit)

Effectue toutes les opérations arithmétiques et logiques : ADD, SUB, AND, ORR, EOR, CMP... Reçoit deux opérandes (depuis le Barrel Shifter et le B-Bus) et produit un résultat + met à jour les flags NZCV.

Résultat → Data Out Register (pour STR en mémoire) OU → ALU Bus (pour écriture dans Register Bank).
🎛️

Instruction Decoder & Control Logic

Décode les instructions 32 bits (ARM) ou 16 bits (Thumb). Génère tous les signaux de contrôle :

  • Sélection des registres Rm, Rn, Rd
  • Opération ALU à effectuer
  • Activation du Barrel Shifter
  • Contrôle du B-Bus
  • Écriture mémoire ou registre
🚌

Les 3 bus internes

BusRôle
A-BusRm vers Barrel Shifter
B-BusRn vers ALU (2ᵉ opérande)
ALU-BusRésultat ALU → Register Bank
Data Bus D(31:0) : bus externe pour accès mémoire (Load/Store). Seuls LDR/STR l'utilisent.
🔁

Pipeline ARM 3 étages — Fetch / Decode / Execute

1. FETCH
L'instruction est lue depuis la mémoire à l'adresse PC. Le PC est automatiquement incrémenté (+4 ARM, +2 Thumb). Implique l'Address Register + Address Incrementer.
2. DECODE
L'Instruction Decoder décode l'opcode et génère les signaux de contrôle. Les registres sources Rm et Rn sont lus depuis la Register Bank.
3. EXECUTE
Le Barrel Shifter pré-traite Rm, l'ALU effectue l'opération, le résultat est écrit dans Rd (Register Bank) ou en mémoire (Data Bus). Les flags NZCV sont mis à jour.
⚡ Avantage pipeline : 3 instructions différentes sont en cours simultanément (F/D/E) → débit théorique de 1 instruction/cycle en régime permanent.
✖️

32×8 Multiplier (Booth Multiplier)

ARM intègre un multiplicateur matériel qui utilise l'algorithme de Booth. Il décompose la multiplication en décalages et additions successifs. Le "32×8" signifie qu'il traite l'opérande de 32 bits par tranches de 8 bits à chaque cycle, permettant de compléter une multiplication en 2–5 cycles selon la valeur de l'opérande.

Instructions concernées : MUL Rd, Rm, Rs · MLA Rd, Rm, Rs, Rn (multiply-accumulate)

SPSR — Saved Program Status Register

Chaque mode d'exception possède son propre SPSR. Lors d'une exception :

1
CPU entre dans le mode d'exception
Le CPSR actuel est sauvegardé dans le SPSR du nouveau mode
2
Traitement de l'exception
Le gestionnaire s'exécute avec ses registres banked dédiés
3
Retour au mode précédent
MOVS PC, LR → restaure CPSR depuis SPSR → reprend le programme interrompu
🔢

Pourquoi les Banked Registers ?

Sans registres banked, une exception interrompant une tâche devrait sauvegarder tous les registres en mémoire avant de s'exécuter (push sur la pile), puis les restaurer au retour. Cela prend de nombreux cycles.

Avec banked registers : pas de push/pop nécessaire pour les registres dédiés → latence d'interruption dramatiquement réduite, critique pour les systèmes temps-réel.

📖

Rôles détaillés R0–R15 (depuis le polycopié)

R0–R12 — Registres généraux
Stockent valeurs temporaires, pointeurs mémoire. R0 = accumulateur (résultat d'opérations arithmétiques). R7 = numéro de syscall. R11 = frame pointer (suivi des limites de pile).
Convention d'appel ARM : Les 4 premiers arguments d'une fonction sont passés dans R0, R1, R2, R3. La valeur de retour est dans R0.
R13 — SP (Stack Pointer)
Pointe vers le sommet de la pile. Pour allouer N octets sur la pile : SUB SP, SP, #N. Pour désallouer : ADD SP, SP, #N. Pour une valeur 32 bits : SUB SP, SP, #4.
R14 — LR (Link Register)
Lors d'un BL label, l'adresse de retour (PC+4 ou PC+2) est automatiquement sauvegardée dans LR. Le retour se fait avec MOV PC, LR ou BX LR.
R15 — PC (Program Counter)
ARM mode : incrémenté de 4 par cycle. Thumb mode : incrémenté de 2. En Thumb, PC est aligné sur des demi-mots (16 bits).

Q·01Que fait l'instruction SUBS par rapport à SUB ?
Le suffixe S (Set flags) signifie que l'instruction met à jour les flags du CPSR. Sans S → pas de mise à jour des flags → les branchements conditionnels ne peuvent pas utiliser le résultat.
Q·02BNE branche si :
BNE = Branch if Not Equal = branche si le résultat n'était pas zéro → Z = 0. Si Z = 1 (résultat était 0), on ne branche pas.
Q·03L'architecture ARM est basée sur quel modèle ?
ARM = Advanced RISC Machine. RISC = Reduced Instruction Set Computer. Caractéristiques : instructions simples, Load/Store, registres nombreux, longueur fixe.
Q·04Combien de registres programmer-visibles ARM possède-t-il ?
ARM possède 16 registres visibles : R0–R12 (généraux), R13 (SP), R14 (LR), R15 (PC).
Q·05Quel est l'avantage principal de l'architecture Harvard par rapport à Von Neumann ?
Harvard = bus séparés pour données et instructions. Permet de lire une instruction ET des données en même temps. Von Neumann doit choisir l'un ou l'autre à chaque cycle (bottleneck).
Q·06Que fait LSL R0, R0, #1 si R0 = 4 (0b0100) ?
LSL #1 = ×2. 0b0100 décalé gauche de 1 = 0b1000 = 8. LSL vers la gauche multiplie par 2 à chaque décalage.
Q·07Pourquoi le DHT22 nécessite-t-il une résistance pull-up ?
Protocole open-drain : le capteur peut tirer la ligne vers GND mais ne peut pas la mettre à VCC seul. La résistance pull-up (10kΩ) assure l'état HAUT par défaut.
Q·08Quelle famille ARM est utilisée dans les microcontrôleurs embarqués ?
Cortex-M = Microcontrollers. Utilise Thumb-2 uniquement. Conçu pour systèmes embarqués bas-consommation (IoT, STM32...). Cortex-A pour smartphones, Cortex-R pour temps-réel critique.
Q·09Quelle instruction transfère des données de la mémoire vers un registre ?
LDR = Load Register : Mémoire → Registre. STR = Store Register : Registre → Mémoire. MOV transfère entre registres ou charge un immédiat (pas de mémoire).
Q·10Le flag Z est mis à 1 après SUBS quand...
Z = 1 uniquement quand le résultat de l'opération est 0. C'est la condition de sortie d'une boucle de décompte : quand le compteur atteint 0, Z=1, BNE ne branche plus.
PIÈGES & MÉMOS

Erreurs fréquentes à éviter

⚠️

Erreurs classiques

  • Confondre ADD (no flags) et ADDS (flags) → les branchements conditionnels ne fonctionneront pas après ADD
  • Oublier que BNE teste Z=0, pas Z=1
  • Penser que MOV accède à la mémoire → Non ! Utiliser LDR/STR pour la mémoire
  • Omettre la résistance pull-up 10kΩ sur la ligne DATA du DHT22
  • Lire le DHT22 sans délai de 2 secondes minimum
  • Confondre LSL (×2ⁿ) et LSR (÷2ⁿ)
✅

Astuces mémo

  • Suffixe S = "Set flags" : SUBS, ADDS, MOVS, LSLS...
  • BNE = "Tant que ≠ 0" → parfait pour les boucles de décompte
  • LSL #n = multiplication par 2ⁿ (optimisation matérielle)
  • Harvard = Arduino AVR = bus séparés = plus rapide
  • Cortex-M = Thumb-2 uniquement = microcontrôleurs
  • R13=SP, R14=LR, R15=PC → les 3 registres spéciaux à retenir
⚠️ Point critique examen : Les instructions sans le suffixe S (MOV, ADD, SUB...) ne modifient PAS les flags NZCV. Toujours vérifier si le branchement conditionnel qui suit pourra fonctionner !
💡 Conseil final : Pour les boucles, le trio magique est MOV (init) + SUBS (décrémente + met Z) + BNE (teste Z). Maîtrisez ce pattern et vous avez la moitié de l'exercice 1 !
